Documentacion del proyecto CRUD de empleados

Esta app fue construida como una demo sencilla con React Native y Expo para que los estudiantes entiendan como una aplicacion movil consume una API REST real y realiza operaciones CRUD.

1. Objetivo del proyecto

El proyecto muestra un ejemplo basico de una aplicacion movil que lista empleados, crea nuevos registros, edita informacion existente y elimina empleados desde una API real.

La idea pedagogica es que el estudiante pueda seguir el flujo sin una arquitectura compleja, sin Redux, sin TypeScript y sin autenticacion.

2. Tecnologias usadas

Frontend movil

React Native

Se usa para construir la interfaz movil con componentes como View, Text y FlatList.

Entorno de desarrollo

Expo SDK 54

Permite ejecutar la app mas facilmente en Expo Go y simplifica el arranque del proyecto.

Navegacion

React Navigation

Se utiliza para moverse entre las pantallas de lista, crear y editar empleado.

Consumo de API

fetch

La comunicacion HTTP se hace con fetch para mantener el codigo simple y facil de aprender.

3. API que consume la app

La aplicacion trabaja con la siguiente URL base:

https://nodejs-test-tfzg.onrender.com/api/empleados

Endpoints utilizados:

  • GET /api/empleados: consulta todos los empleados.
  • POST /api/empleados: crea un nuevo empleado.
  • GET /api/empleados/:id: consulta un empleado por identificador.
  • PUT /api/empleados/:id: actualiza un empleado.
  • DELETE /api/empleados/:id: elimina un empleado.

Campos principales del empleado:

  • _id
  • name
  • position
  • office
  • salary

4. Estructura del proyecto

App.js

Configura la navegacion principal y registra las tres pantallas de la aplicacion.

src/services/api.js

Centraliza la URL base y las funciones que hacen las peticiones HTTP a la API.

src/screens

Contiene las pantallas principales: listar empleados, crear empleado y editar empleado.

src/components

Incluye componentes reutilizables como el formulario, la tarjeta de empleado y los mensajes.

5. Pantallas del proyecto

EmployeeListScreen

Muestra el listado de empleados en una FlatList, permite refrescar, editar y eliminar.

CreateEmployeeScreen

Usa el formulario reutilizable para enviar un POST y crear un empleado nuevo.

EditEmployeeScreen

Puede cargar el detalle del empleado con GET by id y luego actualizarlo con PUT.

6. Componentes reutilizables

EmployeeForm

Se reutiliza tanto para crear como para editar. Maneja los campos name, position, office y salary.

EmployeeItem

Representa cada empleado como una tarjeta visual con botones para editar y eliminar.

MessageBox

Muestra mensajes sencillos de exito o error para que el usuario entienda lo que ocurrio.

7. Flujo general de la app

  1. La app inicia en la pantalla de lista de empleados.
  2. La pantalla consulta la API con GET /api/empleados.
  3. El usuario puede tocar el boton para agregar un nuevo empleado.
  4. El formulario envia un POST para guardar el registro.
  5. Si el usuario edita un registro, la app usa GET /api/empleados/:id si necesita cargar el detalle y luego envia un PUT.
  6. Si el usuario elimina un empleado, la app envia un DELETE y actualiza la lista local.

8. Puntos pedagogicos importantes

  • La logica de la API esta separada en un solo archivo para que sea mas facil de leer.
  • Las pantallas tienen una sola responsabilidad principal.
  • Los componentes se reutilizan para evitar codigo repetido.
  • Los nombres del proyecto coinciden con el dominio real de empleados.
  • La estructura es pequena y suficiente para explicar un CRUD real a estudiantes principiantes.
